TagSpeak — Packet Speak DSL

Version: v1.0.x (active)

Tagline: A tiny packet-based language for human ↔ AI workflows. Calm by design.

What is TagSpeak?

TagSpeak is a symbolic, dataflow-oriented DSL where everything is a packet of the form [op@arg]. Packets chain left → right with >, values flow through, and blocks group work with { ... }.

  • Everything is a packet → [op@arg]
  • Packets chain with > → [math@2+2] > [store@x] > [print@x]
  • Blocks use { ... } to group sequences
  • Strings use quotes → [print@"hello"]
  • Comments welcome: #, //, /* ... */, or TagSpeak’s own [note@"..."]

Quickstart Syntax

Tagspeak is Data flow first. From action to data.

[print@"Hello, TagSpeak!"]

Example Tagspeak code


      [note@ Define count and print initial]

      [int@0]>[store@count]>[print@count]

      

      [note@ Define function called count5]

      

      [funct:count5]{

        [math@count+1]>[store@count]>[print@count]

        

      }[loop:count5@6]

Expected outcome: 0-5 printed to a console

Safety, Consent & Sandbox

  • Red Box (project boundary): put a red.tgsk at repo root. All [load], [save], [log], [cd] are sandboxed to this root; missing root ⇒ E_BOX_REQUIRED.
  • Yellow / Confirm: gate side-effects. [yellow@"why"]{ [exec@"cmd"] } prompts once (or respect env/TOML allowlists).
  • Red Mode: opt-in, ritual acknowledgment; unlocks red-only packets (e.g., [repl]). Red never bypasses Yellow.
  • Non‑interactive: TAGSPEAK_NONINTERACTIVE=1 disables prompts (ops default‑deny unless allowed).

Packet Reference (Canonical)

Canonical packets shipped with TagSpeak, grouped by domain. Sugar forms (like ==) have explicit packet equivalents (like [eq]).

Canonical max form: [packet:label(conditional/argument)@value]

Everything sugars down from there.

Contributing New Packets

Keep the grammar small and behavior composable. Prefer sugar layered over canonical forms. Safety gates are opt‑in and explicit.

  • Avoid duplicates; verify in docs/PACKETS.json via [reflect(packets_full)].
  • Specify inputs/outputs and side‑effects; pass a value through unless explicitly a sink.
  • Gate file/exec/network via red.tgsk, [yellow], and .tagspeak.toml.
  • Document a canonical example and any sugar.
